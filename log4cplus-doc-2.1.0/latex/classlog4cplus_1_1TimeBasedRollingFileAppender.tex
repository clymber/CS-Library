\doxysubsection{Detailed Description}
\mbox{\hyperlink{classlog4cplus_1_1TimeBasedRollingFileAppender}{Time\+Based\+Rolling\+File\+Appender}} extends {\ttfamily \mbox{\hyperlink{classlog4cplus_1_1FileAppenderBase}{File\+Appender\+Base}}} so that the underlying file is rolled over at a user chosen frequency while also keeping in check a total maximum number of produced files. 

\doxyparagraph*{Properties}

Properties additional to {\ttfamily \mbox{\hyperlink{classlog4cplus_1_1FileAppenderBase}{File\+Appender\+Base}}}\textquotesingle{}s properties\+:


\begin{DoxyDescription}
\item[{\ttfamily Filename\+Pattern} ]The mandatory file\+Name\+Pattern property defines the name of the rolled-\/over (archived) log files. Its value should consist of the name of the file, plus a suitably placed d conversion specifier. The d conversion specifier may contain a date-\/and-\/time pattern as specified by the java\textquotesingle{}s Simple\+Date\+Format. The rollover period is inferred from the value of file\+Name\+Pattern.


\item[{\ttfamily Max\+History} ]The optional max\+History property controls the maximum number of archive files to keep, deleting older files.


\item[{\ttfamily Clean\+History\+On\+Start} ]If set to true, archive removal will be executed on appender start up. By default this property is set to false. 


\item[{\ttfamily Roll\+On\+Close} ]This property specifies whether to rollover log files upon shutdown. By default it\textquotesingle{}s set to {\ttfamily true} to retain compatibility with legacy code, however it may lead to undesired behaviour as described in the github issue \#120.


\end{DoxyDescription}
